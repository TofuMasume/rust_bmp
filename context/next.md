## Next Implementation Step

Implement real pixel data generation.

Recommended shape:

```rust
fn pixel_data(&self) -> Vec<u8> {
    let mut bytes = Vec::new();

    for y in (0..self.height).rev() {
        for x in 0..self.width {
            let rgb = self.get_pixel(x, y).expect("coordinates should be in bounds");

            bytes.push(rgb.b);
            bytes.push(rgb.g);
            bytes.push(rgb.r);
        }

        for _ in 0..Image::row_padding(self.width) {
            bytes.push(0);
        }
    }

    bytes
}
```

Then update `to_bytes()` to append `self.pixel_data()` instead of using `resize`.

```rust
pub fn to_bytes(&self) -> Vec<u8> {
    let mut bytes = Vec::new();

    bytes.extend_from_slice(&Image::file_header(self.width, self.height));
    bytes.extend_from_slice(&Image::dib_header(self.width, self.height));
    bytes.extend_from_slice(&self.pixel_data());

    bytes
}
```

## Tests To Add Next

Start with the smallest cases.

1. Verify BGR byte order:

```rust
#[test]
fn to_bytes_writes_pixel_in_bgr_order() {
    let mut image = Image::new(1, 1);
    let red = Rgb { r: 255, g: 0, b: 0 };

    image.set_pixel(0, 0, red).unwrap();

    let bytes = image.to_bytes();

    assert_eq!(&bytes[54..57], &[0, 0, 255]);
}
```

## BMP Details To Keep In Mind

- Pixel data starts at byte offset `54`.
- 24bit BMP stores each pixel as `B, G, R`.
- Rows are padded to a 4-byte boundary.
- Positive height in the DIB header means bottom-up storage, so write rows from `height - 1` down to `0`.
